# Clinica Sorriso Feliz Dashboard

Production dashboard for the WhatsApp AI Agent.

Stack:

- Next.js 15
- TypeScript
- Tailwind CSS
- Supabase Auth, Database, and Realtime
- Recharts

No mock data is used. Every page reads from Supabase tables generated by the WhatsApp AI Agent workflow.

## Features

- Dashboard metrics
- Total conversations
- Total messages
- Conversations by day chart
- Intent distribution chart
- Status distribution chart
- Conversation list
- Search by phone
- Search by keyword across conversation fields and message body
- Conversation detail transcript
- Conversation metrics panel
- Supabase Auth login
- Supabase Realtime updates for `conversations`, `messages`, and `conversation_metrics`

Supported statuses:

- `em_andamento`
- `aguardando_humano`
- `encerrada`

Supported intents:

- `AGENDAMENTO`
- `ORCAMENTO`
- `DUVIDA`
- `HUMANO`

## Environment Variables

Create `.env.local`:

```text
NEXT_PUBLIC_SUPABASE_URL=https://your-project.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=your-supabase-anon-key
```

The dashboard does not need the Supabase service role key. The service role key belongs only in n8n server-side workflows.

## Supabase Setup

Run these SQL files in Supabase:

1. `supabase/migrations/001_ai_whatsapp_agent_schema.sql`
2. `supabase/migrations/002_enable_realtime_for_dashboard.sql`

Create at least one dashboard user in Supabase Auth:

1. Open Supabase Dashboard.
2. Go to Authentication -> Users.
3. Create a user with email and password.
4. Use that account on `/login`.

The schema enables RLS and allows authenticated users to read:

- `conversations`
- `messages`
- `conversation_metrics`

Writes are expected to come from n8n using the Supabase service role key.

## Pages

- `/` Dashboard with totals and Recharts visualizations.
- `/conversations` searchable conversation inbox.
- `/conversations/[id]` full conversation transcript and metrics.
- `/login` Supabase Auth login.

## Realtime Behavior

Client components subscribe to Supabase Realtime and call `router.refresh()` when rows change.

Subscriptions:

- Dashboard and conversation list listen to all changes on `conversations`, `messages`, and `conversation_metrics`.
- Conversation detail listens globally to `conversations` and only message/metric changes for the current `conversation_id`.

## Credential Stop Point

Actual Supabase credentials are required only when running the app:

- `NEXT_PUBLIC_SUPABASE_URL`
- `NEXT_PUBLIC_SUPABASE_ANON_KEY`

Do not put service role credentials in this dashboard.
